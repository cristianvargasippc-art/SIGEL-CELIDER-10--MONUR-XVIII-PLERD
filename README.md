# SISTEMA SIGEL CELIDER 10

Aplicacion web para evaluacion de delegados, consulta publica de resultados, carga de listados Excel, administracion por comision y auditoria de movimientos.

## Base De Datos

Este proyecto queda configurado para **Supabase PostgreSQL**.

No uses SQL de MySQL en Supabase. Por eso fallaba `AUTO_INCREMENT`: Supabase usa PostgreSQL y ahi el equivalente correcto es:

```sql
id INTEGER GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY
```

El SQL listo para Supabase esta en:

```text
backend/prisma/supabase.sql
```

## Flujo De Uso

1. El superadmin inicia sesion.
2. En `Importar`, sube el listado `.xlsx` de delegados.
3. En `Usuarios`, asigna un usuario administrador a cada comision y define su contrasena inicial.
4. Cada admin califica solo su comision.
5. El superadmin revisa dashboard, graficos, auditoria y ranking.
6. Cuando todo este validado, activa `Publicar resultados`.
7. El delegado busca su nombre y ve nombre, delegacion, comision, criterios evaluados, puntaje por criterio, ponderada, mencion y feedback.

## Formato Del Excel

La primera hoja debe tener estos encabezados en la fila 1:

```text
Nombre | Delegacion | Comision
```

Tambien se aceptan encabezados con acento:

```text
Nombre | Delegación | Comisión
```

Ejemplo:

```csv
Nombre,Delegacion,Comision
Ana Isabel Duarte,Republica Dominicana,Asamblea General
Carlos Mendez,Francia,Consejo de Seguridad
Sofia Batista,Brasil,Derechos Humanos
```

## Supabase: Crear Las Tablas

1. Entra a Supabase.
2. Abre tu proyecto.
3. Ve a `SQL Editor`.
4. Abre el archivo `backend/prisma/supabase.sql`.
5. Copia todo el contenido.
6. Pegalo en Supabase SQL Editor.
7. Ejecuta `Run`.

Si ya habias intentado correr el SQL de MySQL y fallo en `AUTO_INCREMENT`, no pasa nada: no se crearon las tablas. Ejecuta ahora el SQL de `backend/prisma/supabase.sql`.

## Supabase: DATABASE_URL Correcta

En Supabase ve a:

```text
Project Settings > Database > Connection string
```

Usa la conexion tipo **URI**.

Debe verse parecido a esto:

```env
DATABASE_URL=postgresql://postgres.TU_PROJECT_REF:TU_PASSWORD@aws-0-us-east-1.pooler.supabase.com:6543/postgres?pgbouncer=true
```

Tambien puede servir la conexion directa:

```env
DATABASE_URL=postgresql://postgres:TU_PASSWORD@db.TU_PROJECT_REF.supabase.co:5432/postgres
```

Recomendacion para hosting: usa la conexion con **pooler** puerto `6543`.

Si tu password tiene simbolos como `@`, `#`, `%`, `/`, espacios o `:`, cambia la contrasena de la base en Supabase por una con letras y numeros para evitar errores en la URL.

SUPERADMIN_EMAIL=admin@tudominio.com

No subas `.env` al repositorio.

## Crear El Superadmin

Despues de crear las tablas en Supabase, ejecuta desde la carpeta `backend`:

```bash
npm install
npx prisma generate
npm run seed
```

Eso crea:

- Comisiones iniciales de ejemplo.
- Configuracion `publish_status=false`.
- Usuario superadmin usando `SUPERADMIN_EMAIL` y `SUPERADMIN_PASSWORD`.

## Deploy En Hostinger Usando Supabase

### 1. Backend Node.js

Sube la carpeta `backend/` a una ruta del hosting, por ejemplo:

```text
/home/u123456789/domains/tudominio.com/backend
```

En `Node.js` de Hostinger configura:

```text
Node version: 20.x o superior
Application root: /home/u123456789/domains/tudominio.com/backend
Application URL: https://api.tudominio.com
Startup file: src/server.js
```

En variables de entorno del panel Node.js coloca:

```env
NODE_ENV=production
PORT=3000
APP_URL=https://tudominio.com
DATABASE_URL=postgresql://postgres.TU_PROJECT_REF:TU_PASSWORD@aws-0-us-east-1.pooler.supabase.com:6543/postgres?pgbouncer=true
JWT_SECRET=CAMBIA_ESTO_POR_UN_TEXTO_LARGO_DE_64_CARACTERES_MINIMO
BCRYPT_ROUNDS=10
SUPERADMIN_EMAIL=admin@tudominio.com
SUPERADMIN_PASSWORD=UnaContrasenaSegura123
```

### 2. Comandos Por SSH

En SSH:

```bash
cd /home/u123456789/domains/tudominio.com/backend
npm install
npx prisma generate
npm run seed
```

No necesitas `npx prisma db push` si ya ejecutaste `backend/prisma/supabase.sql` en Supabase.

Si prefieres que Prisma cree las tablas en vez del SQL manual, puedes usar:

```bash
npx prisma db push
```

Pero no hagas las dos cosas a la vez si no sabes en que estado quedo la base. Lo mas claro para ti ahora es usar el SQL de Supabase y luego `npm run seed`.

### 3. Probar API

Reinicia la app Node.js desde Hostinger y abre:

```text
https://api.tudominio.com/api/health
```

Debe responder:

```json
{"status":"OK","app":"SIGEL CELIDER 10"}
```

### 4. Frontend

Desde la raiz del proyecto:

```bash
npm install
npm run build --workspace frontend
```

Sube todo el contenido de:

```text
frontend/dist
```

a:

```text
public_html
```

o al directorio publico del subdominio donde ira el sistema.

## Prueba Final

1. Abre `https://api.tudominio.com/api/health`.
2. Abre el sistema publico.
3. Entra como superadmin con `SUPERADMIN_EMAIL` y `SUPERADMIN_PASSWORD`.
4. Sube el Excel.
5. Crea usuarios administradores con contrasena inicial.
6. Asigna cada admin a su comision.
7. Registra una calificacion de prueba.
8. Revisa dashboard, graficos y auditoria.
9. Exporta Excel general.
10. Activa `Publicar resultados`.
11. Busca un delegado por nombre en la pantalla publica.

## Criterios De Evaluacion

- Oratoria: 0-15 puntos, peso 15%.
- Argumentacion: 0-25 puntos, peso 25%.
- Negociacion: 0-20 puntos, peso 20%.
- Liderazgo: 0-15 puntos, peso 15%.
- Redaccion: 0-25 puntos, peso 25%.

La calificacion ponderada se normaliza sobre 100 puntos usando el maximo de cada criterio y su peso porcentual.
